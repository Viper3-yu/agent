# Archive 项目库

> 已完成、已放弃、已停摆超过 6 个月的项目。**只读**——仅作历史参考。

## 归档触发

| 触发 | 含义 |
|------|------|
| 项目目标达成 | 完整跑通，转为长期维护 |
| 主动放弃 | 价值不达预期 / 资源不足 / 失去兴趣 |
| 长期停摆 | 6+ 个月无 commit 且无明确复启动机 |
| 外部环境变化 | 依赖 API 停服 / 上游废弃 / 法规禁止 |

## 归档流程

```
1. 在项目 README 顶部加一行：
   > ⚠️ 本项目已于 YYYY-MM-DD 归档，状态：<达成/放弃/停摆>
2. git commit "chore(projects): 归档 <项目名>"
3. git mv 05-Projects/Active/<项目名> 05-Projects/Archive/<项目名>
4. 在本 README "历史归档" 区域加一行
```

## 归档的 4 个理由（老大可裁剪）

1. **保留过程** —— 日后翻看时能看到决策当时的上下文

3. **避免重蹈覆辙** —— 失败的方案可作反面教材
4. **生态贡献** —— 部分老项目可开源 / 写博客
