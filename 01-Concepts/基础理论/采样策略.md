# 采样策略（Sampling Strategies）

> LLM 在每个位置输出一个 logits 向量，采样策略决定如何从这个分布中**挑出下一个 token**。选错策略，要么输出死板重复，要么胡说八道。

## 核心思想

模型最后一层输出 `logits` 向量（vocab size 个分数），转成概率后**采样**出下一个 token：

```
logits [vocab_size]
   ↓ softmax
probs [vocab_size]    # 每个位置是 0~1，所有加起来 = 1
   ↓ 采样策略
selected_token_id
```

**为什么需要采样？**
- 永远选概率最高的 token（Greedy）→ 输出重复、单调
- 完全随机均匀采样 → 输出混乱、无意义
- **折中**：在概率高的 token 中按概率分布采样

## 工作原理

### 1. Greedy / Argmax（贪心）

```
next_token = argmax(probs)
```

- 优点：确定性强、速度快
- 缺点：陷入循环、缺乏多样性
- 适用：分类、抽取、确定性任务

### 2. Temperature Sampling（温度采样）

调整概率分布的"陡峭程度"：

```
probs = softmax(logits / T)
```

- `T → 0`：趋近 Greedy（确定性）
- `T = 1`：原始分布（默认）
- `T → ∞`：趋近均匀分布（完全随机）
- 推荐范围：`0.0 ~ 1.5`

```
T = 0.2 → "保守、精确"（代码、事实问答）
T = 0.7 → "平衡"（聊天、问答）
T = 1.0 → "原始"
T = 1.2 → "创造性"（写作、头脑风暴）
```

### 3. Top-k Sampling

**只从概率最高的 K 个 token 中采样**：

```
1. 取概率最高的 K 个 token
2. 重新归一化这 K 个概率
3. 在这 K 个中按概率采样
```

- `k = 1`：等价于 Greedy
- `k = 40`：常见默认（GPT-3）
- 缺点：分布平的时候（前 40 个概率差不多），K 太死板

### 4. Top-p (Nucleus) Sampling

**从累积概率达到 p 的最小 token 集合中采样**：

```
1. 按概率降序排列 token
2. 找到累积概率 ≥ p 的最小集合
3. 在该集合中重新归一化后采样
```

- `p = 0.1`：只考虑前 10% 概率
- `p = 0.9`：覆盖 90% 概率（常用）
- `p = 1.0`：全 vocab

**优势**：自适应——分布尖时少取、分布平时多取。比 Top-k 更鲁棒。

### 5. Min-p Sampling（2024 流行）

**只考虑概率 ≥ min_p × max_prob 的 token**：

```
threshold = min_p × max(probs)
mask = probs >= threshold
```

- 与 top 模型概率**成比例**缩放
- 强模型（max_prob 高）→ 阈值高 → 取少
- 弱模型（max_prob 低）→ 阈值低 → 取多
- 更适合推理模型（GPT-o1、Claude Thinking）

### 6. Beam Search

**保留 top-B 个候选序列，每步扩展**：

```
Step 1: ["I", "The", "Here"]  (B=3)
Step 2: ["I am", "I was", "The cat", ...]  (按累积概率保留 B=3)
Step 3: ... 直至 EOS
```

- 优点：全局最优的概率序列
- 缺点：多样性差、计算量 B 倍
- 适用：翻译、ASR 等需要"最优解"的任务
- **不适用于生成式 LLM**：会输出"最无聊"的安全回答

## 推理 API 常用参数

### OpenAI

```python
response = client.chat.completions.create(
    model="gpt-5.5",
    temperature=0.7,        # 0.0 ~ 2.0
    top_p=0.9,              # 0.0 ~ 1.0
    frequency_penalty=0.0,  # -2.0 ~ 2.0，减少重复
    presence_penalty=0.0,   # -2.0 ~ 2.0，鼓励新话题
    logit_bias={...},       # 给特定 token 加/减权重
)
```

### Anthropic Claude

```python
response = client.messages.create(
    model="claude-opus-4-8",
    temperature=1.0,        # 默认 1.0
    top_p=0.95,             # Anthropic 推荐 0.95
    top_k=40,               # 可选
)
```

### Google Gemini

```python
generation_config = {
    "temperature": 0.9,
    "topP": 1.0,
    "topK": 40,
    "candidateCount": 1,    # 生成几个候选
}
```

## 推理模型 vs 对话模型的采样差异

| 模型类型 | 推荐策略 |
|---------|---------|
| 对话 / 创作 | `temperature=0.7-1.0`、`top_p=0.9` |
| 代码生成 | `temperature=0.0-0.2`（追求确定性） |
| 抽取 / 分类 | `temperature=0.0`（Greedy） |
| 推理（o1, Claude Thinking） | **内部固定**，不支持外部调参 |
| 数学证明 | `temperature=0.0` + 多采样投票 |
| 翻译 | Beam search 或 `temperature=0.3` |

## Repetition Penalty（重复惩罚）

降低已生成 token 的概率：

```
logits[token] -= α   if token 已出现过
```

- `α = 0`：无惩罚
- `α = 0.3`：轻度防重复（对话常用）
- `α = 1.0`：激进防重复（可能语义扭曲）

## DRY / Anti-Sycophancy（2025+）

针对"AI 拍马屁"问题（用户说错也附和），新引入的策略：

- **DRY Sampler**：检测"赞美词"模式，主动降权
- **Anti-Sycophancy**：检测用户措辞中的情绪，触发冷静分支

## 优势与局限

### 优势
- 让 LLM 输出可控（确定性 vs 创造性）
- 多个 API 参数可组合微调
- 推理模型自动选择最优策略
- 重复惩罚 / 话题惩罚解决简单质量问题

### 局限
- **没有万能组合**：不同模型对同一组参数响应不同
- **指标难量化**：质量评估主观，无法搜索最优超参
- **API 默认值未必最优**：Anthropic 默认 `temp=1.0` 对许多任务偏高
- **推理模型不可控**：o1 / Claude Thinking 锁死采样，无法外部干预
- **多模态采样更复杂**：图像 / 音频 token 的采样策略尚未标准化
