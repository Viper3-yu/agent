# 对齐与安全（Alignment & Safety）

> 确保 AI 系统的行为、目标和价值观与人类意图、伦理和社会规范保持一致。是模型从"能做事"到"做正确的事"的关键步骤。

## 核心思想

**问题陈述**：模型能力越强，未对齐的风险越大。三类典型风险：

1. **目标错位**（Outer Alignment）—— 损失函数没有完全捕获人类意图
2. **可解释性缺失**（Inner Alignment）—— 不知道模型内部在学什么 mesa-optimizer
3. **滥用与越狱**（Misuse）—— 用户故意诱导模型输出有害内容

**对齐方法 = 三阶段流水线 + 持续评估**：

```
   海量文本
      ↓
   [1] 预训练 Pretrain  ← 获得通用能力
      ↓
   [2] 监督微调 SFT    ← 学会"怎么回答"
      ↓
   [3] 偏好对齐        ← 学会"什么是好答案"
       ├─ RLHF（PPO）
       ├─ DPO
       ├─ GRPO（DeepSeek-R1）
       ├─ Constitutional AI / RLAIF
       └─ 在线 RL（o1 / Claude Opus 4 推理增强）
      ↓
   [4] 评估 + 红队  ← 持续检测漏洞
```

## 工作原理

### 1. SFT（Supervised Fine-Tuning）

- 收集数千~数万条人类示范回答
- 标准语言建模 loss 微调
- 让模型学会"对话格式"和"基础能力"

### 2. RLHF（Reinforcement Learning from Human Feedback）

三步法（InstructGPT 论文范式）：

```
1. 收集人类偏好
   prompt → 模型生成 A/B → 人类标注哪个更好
   
2. 训练奖励模型 RM
   (prompt, A, B) → RM 评分 → 拟合人类偏好
   
3. PPO 强化学习
   模型生成 → RM 打分 → PPO 更新
```

**问题**：训练不稳定、RM 难校准、计算贵。

### 3. DPO（Direct Preference Optimization）

跳过 RM，直接用偏好数据优化策略：

```
L_DPO = -log σ(β · log(π_θ(y_w|x)/π_ref(y_w|x))
                - β · log(π_θ(y_l|x)/π_ref(y_l|x)))
```

其中 y_w 是偏好回答，y_l 是非偏好回答。**更简单、更稳、效果近似 RLHF**。

### 4. GRPO（Group Relative Policy Optimization）

DeepSeek-R1 提出的变体：
- 对每个 prompt 采样一组回答
- 用**组内相对奖励**（不需要单独 critic/RM）替代绝对奖励
- 适合推理任务

### 5. Constitutional AI / RLAIF

- 用一组"宪法"原则（如"不伤害人"、"不撒谎"）作为规则
- 让另一个 LLM 评判并修改回答（RLAIF = RL from AI Feedback）
- 减少人类标注成本

### 6. 推理时对齐

- **o1 / Claude Opus 4 思考模式**：训练时让模型学会长 CoT，推理时显式推理
- **System Prompt / Constitutional Prompt**：在 prompt 层注入约束
- **工具辅助审核**：用 Moderation API / Llama Guard 等过滤输出

## 优势与局限

### ✅ 已有成就
- **能力对齐**：现代旗舰模型在 MMLU / GPQA 等任务上达人类水平
- **拒绝有害请求**：ChatGPT / Claude 等普遍能拒答
- **推理增强**：o1 / DeepSeek-R1 表明 RL 可激发模型推理能力

### ❌ 仍未解决
- **Jailbreak 攻防**：prompt injection 仍是开放问题
- **幻觉**：模型自信地编造事实
- **长尾风险**：能力越强，misuse 越难防
- **价值对齐**：跨文化、跨群体的价值观难以统一
- **可解释性**：黑盒决策，事后分析为主
- **目标博弈**：mesa-optimization 理论风险
