# 量化（Quantization）

> 把模型权重和激活从 FP32 / FP16 映射到低比特整数（INT8 / INT4 / NF4），让 70B 模型塞进 24GB 消费级 GPU。是 LLM 部署的"最后一公里"。

## 核心思想

```
FP32: 1 个权重 = 4 字节，70B 模型 = 280GB
FP16: 1 个权重 = 2 字节，70B 模型 = 140GB
INT8: 1 个权重 = 1 字节，70B 模型 =  70GB
INT4: 1 个权重 = 0.5 字节，70B 模型 = 35GB
NF4: 1 个权重 = 0.5 字节，70B 模型 = 35GB（质量接近 FP16）
INT2: 1 个权重 = 0.25 字节，70B 模型 = 17.5GB（质量差）
```

**核心矛盾**：比特数越少 → 模型越小、推理越快 → 但**精度损失**越大。量化研究的全部努力都是在**保住质量的前提下压比特数**。

## 工作原理

### 量化公式（对称量化）

```
scale = (max - min) / (qmax - qmin)
zero_point = qmin - round(min / scale)

quantize(x) = round(x / scale) + zero_point
dequantize(q_x) = (q_x - zero_point) × scale
```

### 粒度（Granularity）

| 粒度 | 范围 | 精度 | 速度 |
|------|------|------|------|
| Per-tensor | 整个层一个 scale/zero_point | 差 | 快 |
| Per-channel | 每个输出通道一个 scale | 中 | 中 |
| Per-group（group_size=128） | 每 128 个权重一个 scale | 好 | 慢 |

**LLM 量化常用 per-group**，平衡精度和速度。

## 主流方案

### 1. PTQ（Post-Training Quantization）

训练**完成后**量化，无需重新训练：

```
预训练 FP32 模型
   ↓ 准备校准数据（~100 条样本）
   ↓ 统计每层激活分布
   ↓ 计算每组最优 scale / zero_point
   ↓ 量化权重
INT8 / INT4 模型
```

优点：成本低（几小时）。
缺点：精度损失稍大，需要校准数据。

### 2. QAT（Quantization-Aware Training）

训练**过程中**模拟量化，让模型适应量化噪声：

```
FP32 训练 + 伪量化算子（fake quant）
   ↓ 梯度回传
   ↓ 模型学会"绕过"量化误差
最终量化模型
```

优点：精度最佳（接近 FP16）。
缺点：训练成本等同从头训练，仅用于自家模型。

### 3. GPTQ（2023 经典）

逐层**最优脑量化**（Optimal Brain Quantization）：

```
逐层处理：
  for layer in model:
    用校准数据计算 Hessian 矩阵 H
    量化权重 w → q_w，最小化 (q_w - w)^T H (q_w - w)
    补偿：未量化的权重 w' = w + H^-1 · grad
```

特点：单 GPU 几小时内量化 70B 模型，**质量接近原始**。

### 4. AWQ（Activation-aware Weight Quantization）

观察到"激活值大的权重更重要"：

```
对每个权重：
  importance_score = |weight| × mean(|activation|)
重要权重保持 FP16
其他权重量化 INT4
```

效果：比 GPTQ 略好，特别是 MoE 模型。

### 5. NF4（4-bit NormalFloat）

QLoRA 使用的格式。**信息论最优的 4-bit 表示**：

```
NF4 量化级别（16 个）按正态分布分位点取：
[-1.0, -0.6961, -0.5251, -0.3949, -0.2844, -0.1848,
 -0.0911, 0.0, 0.0796, 0.1609, 0.2461, 0.3379, 0.4404,
 0.5626, 0.7230, 1.0]
```

特点：模型权重通常近似正态分布，NF4 用最优分位点编码，**精度高于 INT4**。

### 6. GGUF（llama.cpp 格式）

CPU 友好的量化格式，支持多种比特数：

| 比特 | 名称 | 7B 模型大小 | 质量 |
|------|------|-------------|------|
| Q8_0 | 8-bit | 7 GB | ≈ FP16 |
| Q6_K | 6-bit | 5.5 GB | 极接近 FP16 |
| Q5_K_M | 5-bit | 4.8 GB | 推荐平衡点 |
| Q4_K_M | 4-bit | 4.1 GB | **消费级首选** |
| Q3_K_M | 3-bit | 3.3 GB | 可接受 |
| Q2_K | 2-bit | 2.7 GB | 质量明显下降 |

| 场景 | 推荐方案 | 模型大小 | 硬件 |

## 质量损失与校准

### 量化误差

```
error = ||W_quantized · x - W · x|| / ||W · x||
```

- INT8：误差 < 1%，无感
- INT4：误差 2-5%，轻任务无感、复杂推理略降
- INT2：误差 10-20%，明显退化

### 校准数据选择

校准数据应**代表真实部署分布**：
- 聊天 → 用真实对话样本
- 代码 → 用代码样本
- 多语言 → 平衡各语言样本

**陷阱**：用 wiki 校准、用代码推理 → 量化模型在代码上崩。

### 评估指标

量化后必须验证：
- **Perplexity**：在 WikiText 上 PPL 涨幅 < 5%
- **任务指标**：MMLU、HumanEval 等 < 2% 降幅
- **端到端**：在你的真实场景中跑回归

## 与 QLoRA 的关系

**QLoRA = 4-bit 量化 + LoRA 适配器**

详见 [[../06-Resources/微调/LoRA 与 QLoRA|LoRA 与 QLoRA]]：

```
Base Model (NF4 量化，冻结)
   + LoRA Adapter (FP16，可训练)
   = QLoRA 训练
```

- 显存：7B 模型 QLoRA 训练只需 ~10GB（消费级 3090）
- 速度：比 FP16 LoRA 慢 ~30%
- 质量：接近 FP16 LoRA

## 优势与局限

### 优势
- **大幅压缩**：4-bit 让 70B 模型消费级可跑
- **推理加速**：低比特运算在现代 GPU 上更快（INT4 Tensor Core）
- **节能**：内存带宽是推理瓶颈，量化减少数据传输
- **兼容性强**：GGUF/ONNX 等格式生态成熟

### 局限
- **质量损失**：低于 4-bit 退化明显
- **校准敏感**：校准数据分布不匹配 → 模型崩
- **硬件依赖**：INT4 Tensor Core 仅 NVIDIA 4090+ / H100
- **微调困难**：量化后微调需要 PEFT（QLoRA / QAT）
- **MoE 不友好**：专家稀疏激活，激活范围难估计 → 量化误差大
- **黑盒调试**：量化 bug 难定位（输出错误但报错信息少）
