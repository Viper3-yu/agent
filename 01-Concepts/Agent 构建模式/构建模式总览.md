# Agent 构建模式总览

> Anthropic "Building Effective Agents" 提出的核心理念：**先用最简单的方式（裸 API + 提示链），复杂度真正需要时再上框架。**

## 两大类模式

### 基础模式（单 Agent 增强）
- [[Prompt Chaining]] — 顺序串联多个步骤
- [[路由模式 Routing]] — 根据输入分发到不同处理路径
- [[并行化 Parallelization]] — 同时执行多个独立子任务
- [[编排者-工人模式 Orchestrator-Worker]] — 一个 Agent 编排多个工人

### 高级模式（多 Agent 协作）
- [[评估者-优化者模式 Evaluator-Optimizer]] — 一个生成，一个评估，循环改进
- [[Human-in-the-Loop]] — 关键节点引入人工审批

## 适用场景

1. 需要结构化地设计 Agent 工作流时
2. 任务复杂度超出单次 LLM 调用能力时
3. 需要在可靠性、延迟和成本之间做权衡时
4. 从原型走向生产，需要可维护的架构时

## 不适用场景

1. 简单的单轮问答 — 直接调用裸 API + 优化 Prompt 即可
2. 纯规则引擎能解决的确定性问题 — 不需要 LLM 介入
3. 对延迟极度敏感的实时系统 — 多步编排会增加延迟

## 选型原则

```
简单需求 → 裸 API 调用 + Prompt 优化
     ↓ 不够
单步增强 → Prompt Chaining / Routing
     ↓ 不够
多步编排 → Orchestrator-Worker
     ↓ 不够
质量保障 → Evaluator-Optimizer
     ↓ 不够
多 Agent → 框架（LangGraph / CrewAI）
```

> "The most successful agent implementations we've seen in production are often the simplest." — Anthropic

## 模式对比速查表

| 模式 | 核心思想 | 复杂度 | 典型场景 |
|------|----------|--------|----------|
| [[Prompt Chaining]] | 顺序串联 | 低 | 多步处理流程 |
| [[路由模式 Routing]] | 分类分发 | 低 | 多类型输入处理 |
| [[并行化 Parallelization]] | 同时执行 | 中 | 多维度分析、投票 |
| [[编排者-工人模式 Orchestrator-Worker]] | 动态编排 | 中 | 复杂任务分解 |
| [[评估者-优化者模式 Evaluator-Optimizer]] | 生成-评估循环 | 高 | 高质量输出需求 |
| [[Human-in-the-Loop]] | 人工介入 | 中 | 高风险操作审批 |
