# 向量数据库总览

> 存储和检索向量（Embedding）的专用数据库，是 RAG 系统的核心基础设施。

## 核心架构

向量数据库的核心是 ANN（Approximate Nearest Neighbor）算法，在高维空间中快速检索相似向量。主流方案包括 HNSW、IVF、SCANN 等。

### 核心概念

- **ANN 算法**：HNSW（主流）、IVF、SCANN
- **距离度量**：余弦相似度、欧氏距离、内积
- **元数据过滤**：在向量搜索的同时按字段过滤
- **混合搜索**：向量 + 关键词联合检索

## 主流向量数据库

### 托管服务

| 数据库 | 厂商 | 特点 |
|--------|------|------|
| Pinecone | Pinecone | 全托管，零运维，Serverless |
| Weaviate Cloud | Weaviate | 多模态，GraphQL API |
| Qdrant Cloud | Qdrant | 高性能，Rust 编写 |
| pgvector (Supabase) | Supabase | PostgreSQL 扩展，适合已有 PG 的项目 |

### 开源自部署

| 数据库 | 语言 | 特点 |
|--------|------|------|
| Qdrant | Rust | 高性能，丰富过滤 |
| Weaviate | Go | 多模态，模块化 |
| Chroma | Python | 极简 API，适合原型 |
| Milvus | Go/C++ | 大规模，GPU 加速 |
| LanceDB | Rust | 嵌入式，零服务器 |
| pgvector | C | PostgreSQL 扩展 |

## 典型使用场景

1. **RAG 系统**: 存储文档 Embedding，配合检索增强生成
2. **语义搜索**: 基于语义相似度的内容检索
3. **推荐系统**: 基于向量相似度的个性化推荐
4. **多模态检索**: 图像、音频等非文本内容的相似度检索

## 选型建议

| 场景 | 推荐 |
|------|------|
| 快速原型 | Chroma（Python 内嵌） |
| 生产级通用 | Qdrant 或 Weaviate |
| 已有 PostgreSQL | pgvector |
| 超大规模 | Milvus |
| 零运维 | Pinecone Serverless |
| 嵌入式/边缘 | LanceDB |

## 已知局限

- 高维向量的检索精度与速度 trade-off
- 元数据过滤能力因产品差异较大
- 大规模数据的索引构建时间和内存消耗
