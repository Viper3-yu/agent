# Agent 评估框架总览

## 概述

系统化评估 Agent 质量需要多维度指标体系和标准化的评估流程。本文梳理主流评估框架、评估方法和关键维度，为团队建立 Agent 质量保障体系提供方法论参考。

## 评估维度

| 维度 | 指标 | 计算方式 | 参考值 |
|------|------|----------|--------|
| 准确性 | 任务完成率、正确率 | exact_match / F1 / 人工标注 | > 90% |
| 相关性 | 检索相关性（RAG） | RAGAS relevance score | > 0.7 |
| 安全性 | 拒绝率、注入防御 | 红队测试攻击成功率 | < 5% |
| 效率 | Token 成本、延迟 | 日志分析 P50/P95 | 依场景而定 |
| 鲁棒性 | 边界情况处理 | 对抗测试通过率 | > 85% |

## 主流评估框架

| 框架 | 定位 | 特点 |
|------|------|------|
| RAGAS | RAG 评估 | 自动化 RAG 质量评分 |
| AgentBench | Agent 评估 | 多环境 Agent 基准 |
| LMSYS Arena | 人类偏好 | ELO 排名系统 |
| promptfoo | Prompt 评估 | CLI 工具，多模型对比 |
| DeepEval | 通用评估 | 14+ 指标，CI/CD 集成 |

## 使用方法

### 输入格式

```json
{
  "eval_id": "eval-001",
  "test_cases": [
    {
      "input": "用户的查询",
      "expected_output": "期望的回复",
      "context": ["检索到的文档片段"],
      "tags": ["rag", "single-hop"]
    }
  ],
  "metrics": ["relevancy", "faithfulness", "hallucination"]
}
```

### 输出格式

```json
{
  "eval_id": "eval-001",
  "overall_score": 0.87,
  "pass_rate": 0.92,
  "per_metric": {
    "relevancy": { "mean": 0.85, "std": 0.12 },
    "faithfulness": { "mean": 0.90, "std": 0.08 },
    "hallucination": { "mean": 0.05, "std": 0.03 }
  },
  "failed_cases": ["tc-012", "tc-045"]
}
```

### 代码示例

```python
from deepeval import evaluate
from deepeval.metrics import AnswerRelevancyMetric, FaithfulnessMetric
from deepeval.test_case import LLMTestCase

test_case = LLMTestCase(
    input="公司的退款政策是什么？",
    actual_output=agent.run("公司的退款政策是什么？"),
    expected_output="30天内无理由退款。",
    retrieval_context=[retrieved_docs],
)

results = evaluate(
    [test_case],
    [AnswerRelevancyMetric(threshold=0.7), FaithfulnessMetric(threshold=0.8)],
)
```

### 评估方法

- **自动评估**：LLM-as-Judge（用强模型评估弱模型）
- **人工评估**：众包标注、专家评审
- **基准测试**：标准化任务集
- **A/B 测试**：线上对比不同版本

## 适用场景

- RAG 系统的检索和生成质量评估
- Agent 工具调用正确性验证
- Prompt 迭代效果的量化对比
- 模型版本升级前后的回归测试
- CI/CD 流水线中的质量门禁

## 局限性

- LLM-as-Judge 存在位置偏好（Position Bias）和冗长偏好（Verbosity Bias）
- 自动评估与人工评估的一致性通常在 80-90%，仍有偏差
- 评估数据集的构建和维护成本较高
- 不同框架的指标定义和计算方式不统一，跨框架对比困难

## 与其他评估方法的对比

| 特性 | 自动评估框架 | 人工评估 | 可观测性监控 | A/B 测试 |
|------|------------|----------|------------|----------|
| 成本 | 低-中 | 高 | 低 | 中 |
| 速度 | 快（分钟级） | 慢（天级） | 实时 | 慢（周级） |
| 覆盖度 | 全量 | 抽样 | 全量 | 分流 |
| 准确性 | 中-高 | 高 | N/A | 高 |
| 适用阶段 | 开发 + CI | 发布前校准 | 生产 | 线上对比 |
| 可复现性 | 高 | 低 | 高 | 中 |
