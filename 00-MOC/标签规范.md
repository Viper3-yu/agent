# 标签规范 v1

> vault 标签系统的**完整定义和使用规则**。与 [[KB 构建规范]] 互为补充：本文件只管标签，[[KB 构建规范]] 管其他 frontmatter 字段。

---

## 一、三层标签架构

每条笔记的 `tags:` 数组由 **三类组成**，可缺省但建议都填：

```
[1] type        — 笔记类型（互斥，1 个）
[2] topic/xxx   — 主题领域（可多 0~2 个）
[3] status/xxx  — 成熟度（互斥，1 个）
```

**示例**：
```yaml
tags:
  - protocol              # [1] type
  - topic/agent           # [2] topic
  - status/reviewed       # [3] status
```

笔记**类型**，决定它在 vault 哪个目录、用哪个模板。

| Tag | 适用 |
|-----|------|
| `concept` | 01-Concepts/ 下的概念笔记 |
| `model` | 02-Models/ 下的模型笔记 |
| `framework` | 04-Frameworks/ 下的框架 |
| `protocol` | 04-Frameworks/ 下的协议 |
| `product` | 04-Frameworks/Agent 产品/ 下的产品 |
| `pattern` | 01-Concepts/Agent 构建模式/ 下的模式 |
| `paper` | 06-Resources/Papers/ 下的论文笔记 |
| `fine-tuning` | 06-Resources/微调/ 下的方法 |
| `evaluation` | 04-Frameworks/Agent 可观测性/ 下的评估 |
| `prompt` | 03-Prompt/ 下的 prompt 笔记 |

⚠️ **互斥**——一篇笔记只能有 1 个 type（除 MOC 例外）。

## 四、Status 标签（status/*，互斥）

笔记的**成熟度**，决定是否可被信任、是否需要 Claudian 进一步补全。

| Tag | 含义 | 触发 | 当前使用 |
|-----|------|------|---------|
| `status/seed` | 初始占位 | 仅新建空白笔记 / 模板 | 0 篇笔记 + 11 篇模板 |
| `status/draft` | 起草中 | Claudian 起草未审 | — |
| `status/reviewed` | 已审可用 | 老大点头通过 | 127 篇 |
| `status/mature` | 已成熟 | 深填充 + 多链接 + 多次引用 | 12 篇（纯概念笔记：[[../01-Concepts/Agent 构建模式/Prompt Chaining\|Agent 构建模式]] 7 + [[../03-Prompt/技术/Chain of Thought\|CoT]]/[[../03-Prompt/技术/ReAct Prompting\|ReAct Prompting]] 2 + [[../06-Resources/微调/LoRA 与 QLoRA\|LoRA/QLoRA]] 2）|
| `status/archived` | 已归档 | 协议废弃、产品停服 | — |

⚠️ **配套 meta tag**：`meta/stale` —— 标志笔记内容依赖**时间敏感信息**（具体模型/版本/年份），联网核查失败时降 `status/reviewed` 用，**不与 status 互斥**，可同时存在。当前 32 篇（2026-06-13 审计后；详见 [[../Inbox/00-stale-audit-2026-06-13]]）。

### mature 的更严格门槛

`status/mature` 必须是**纯概念/方法论**笔记（不会因时间而失真），例：
- ✅ Anthropic 源 [[../01-Concepts/Agent 构建模式/Prompt Chaining\|Prompt Chaining]] 等模式笔记
- ✅ 通用技术概念（RAG、CoT、ReAct）
- ❌ 具体模型笔记（GPT-5、Gemini 3.x）—— 厂商会迭代
- ❌ 具体微调方法（LoRA 2024 版）—— 领域在演进
- ❌ 具体框架（LangChain v0.3）—— 版本敏感

时间敏感笔记一律停在 `status/reviewed` + `meta/stale`，等联网核查通过或人工确认无变化后再升 mature。

## 五、标签使用铁律

1. **每篇必填 status**——空着就是 `status/seed`
2. **type 必填**——位置对不上目录时 Obsidian 会警告
3. **topic 鼓励填**——0~2 个，不强求
4. **不用裸领域词**——所有领域标签必须 `topic/xxx`
5. **不用单复数混用**——`model` 不是 `models`、`protocol` 不是 `protocols`
6. **不用时间标签**——`status/2026-q2` 这类不允许（用 MOC 时间线笔记代替）
7. **不用临时标签**——`#todo` `#wip` 等任务标签写进笔记正文，别进 frontmatter
